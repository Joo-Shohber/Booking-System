Queue Promotion Flow
❌ Cancellation
  • 1Booking status → cancelled
  • 2slot.booked-- (Mongo txn)
  • 3RELEASE_SLOT_SCRIPT (Redis INCR)
  • 4Enqueue PROMOTION job immediately
🚀 Promotion Worker
  • 1Read available = capacity - booked
  • 2if available ≤ 0 → exit (idempotent)
  • 3promoteBatch(slotId, available)
  • 4Lock queue:{slotId} for entire batch
📢 promoteBatch
  • 1ZPOPMIN queue:{slotId} (FIFO)
  • 2QueueEntry → status: notified
  • 3WS: queue:your_turn + deadline
  • 4Repeat for each available seat
✅ User Confirms
  • 1Within QUEUE_CONFIRM_MINUTES
  • 2BOOK_SLOT_SCRIPT (atomic DECR)
  • 3Mongo txn: slot.booked++ + booking
  • 4QueueEntry → confirmed
Redis Key Schema
Key Pattern Type Purpose
slot:{slotId}:available String DECR/INCR counter — primary overbooking prevention
queue:{slotId} Sorted Set FIFO queue, score = timestamp
queue:length:{slotId} String Cached ZCARD to avoid per-request ZCARD calls
provider:slots:{id}:{date} String (JSON) Cached slot list, TTL = CACHE_TTL_SECONDS
slot:availability:{slotId} String (JSON) Cached single slot enriched DTO
rt:{userId}:{jti} String SHA-256 hashed refresh token, TTL = refresh expiry
rl:{ip}:{path} Sorted Set Sliding window rate limit entries
idem:{userId}:{key} String (JSON) Cached idempotent response, TTL = 24h
lock:queue:{slotId} String Distributed lock for promoteBatch only
Lua Scripts — Atomic Redis Operations
BOOK_SLOT_SCRIPT
local current = redis.call("GET", KEYS[1])
if current == false then
  return -2 -- key missing, init needed
end
if tonumber(current) <= 0 then
  return -1 -- slot full
end
return redis.call("DECR", KEYS[1])
RATE_LIMIT_SCRIPT
redis.call("ZREMRANGEBYSCORE",
  KEYS[1], 0, ARGV[2])
redis.call("ZADD",
  KEYS[1], ARGV[1], ARGV[1])
local count = redis.call(
  "ZCARD", KEYS[1])
redis.call("EXPIRE",
  KEYS[1], ARGV[3])
return count
